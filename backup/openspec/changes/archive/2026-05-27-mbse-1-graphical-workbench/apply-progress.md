## Notes

- `src/mbse_1` now has its own package skeleton and explicit domain contract types, with no imports from legacy `src/mbse`.
- The prepared-document validator treats authored Graphviz `id="..."` attributes as the only accepted highlight contract source, which keeps later frontend/backend work anchored to exact SVG IDs rather than visible text.
- Validation currently raises small local `ValidationError` diagnostics instead of reusing legacy shared diagnostics so the slice remains architecturally isolated.
- Work Unit 2 now proves the exact-ID pipeline end-to-end: prepared JSON -> `dot -Tsvg` -> SVG XML ID extraction -> rendered-ID validation -> `diagram.svg` + `workbench-session.json`.
- Missing rendered IDs are validated against the rendered SVG, not against visible labels or titles, which keeps the frontend/backend contract minimal and unequivocal.
- Work Unit 3 keeps highlighting exact and ordered end-to-end: the backend returns `active_ids` and `unknown_ids` in request order, and `Idle` remains unknown unless that literal SVG ID exists.
- The read-only workbench uses only stdlib HTTP serving plus tiny static assets under `src/mbse_1/web/static`, which avoids importing any legacy viewer/runtime code while still supporting local launch.
- The bounded artifact set now includes the original prepared JSON document, copied into the session directory under its source filename so the session remains self-contained without adding new contract fields.
- Server startup now revalidates persisted `workbench-session.json.highlightable_ids` against the exact IDs extracted from `diagram.svg`, so stale or hand-edited sessions fail fast before any misleading highlight API can start.
- The CLI lifecycle now stays alive by waiting on the running server wrapper instead of relying on the daemon HTTP thread, so printing the local URL no longer terminates the process before the browser can connect.
